# vp-cursor-portal v0.1.0

First MVP release for Windows cursor routing and video processor preset control.

## Assets

- `vp-cursor-portal-setup.exe`: Windows installer that installs under `Program Files`.
- `vp-cursor-portal-win-x64.zip`: self-contained portable Windows x64 build.

## Highlights

- Dashboard-first WPF app for profile execution, routing status, H2 status, safety controls, logs, and diagnostics.
- H2 preset loading over UDP JSON with ACK handling.
- Cursor layout activation after successful preset ACK when the profile requires it.
- Pure cursor routing engine with visible-zone, hidden-zone, outside-zone, full-edge portal, segmented portal, and ratio-mapping behavior.
- Layout editor with detected monitor coordinates, scaled canvas editing, snapping, and portal generation.
- Emergency unlock through the UI and `Ctrl+Alt+Shift+Esc`.
- Tray behavior and Windows startup option.
- Installer uninstall option to remove `%AppData%\vp-cursor-portal`.

## Known Limitations

- The installer and executable are not code-signed yet, so Microsoft Defender SmartScreen may show an unknown publisher warning.
- Routing is polling-based; low-level mouse hook mode is not included in this MVP.
- Video processor layout discovery is not automatic. Users configure cursor layouts explicitly.
- H2 is the first supported device family. Broader video processor support is planned as a future extension.

## Install Notes

For most users, download and run `vp-cursor-portal-setup.exe`.

Use the ZIP artifact only when you want a portable folder without installer registration.

On first launch, routing starts disabled. Create or import devices, layouts, and profiles before enabling routing through a profile.

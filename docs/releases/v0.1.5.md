# vp-cursor-portal v0.1.5

Patch release for Identify overlay placement in mixed-DPI monitor layouts.

## Assets

- `vp-cursor-portal-setup.exe`: Windows installer that installs under `Program Files`.
- `vp-cursor-portal-win-x64.zip`: self-contained portable Windows x64 build.

## Fixes

- Positions Identify overlays with Win32 physical pixels after the WPF window handle is created.
- Supports monitors with negative Windows coordinates, such as displays placed left of or above the primary monitor.
- Improves Identify placement for mixed 4K 150% scaling and FHD 100% scaling field layouts.
- Re-centers Identify windows after WPF layout and DPI initialization so the final window location matches the physical monitor bounds.

## Compatibility

- Cursor routing behavior is unchanged from v0.1.4.
- Portal-only visible-zone transitions, hidden-zone rejection, outside-zone rejection, and ratio-based portal mapping are preserved.

## Validation

- Reviewed the Identify placement path for logical-pixel, physical-pixel, negative-coordinate, and mixed-DPI behavior.
- Confirmed the change is isolated to display identification window placement.
- Run `git diff --check` before release tagging.

## Install Notes

For most users, download and run `vp-cursor-portal-setup.exe`.

Use the ZIP artifact only when you want a portable folder without installer registration.

The installer and executable are not code-signed yet, so Microsoft Defender SmartScreen may show an unknown publisher warning.

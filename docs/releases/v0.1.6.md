# vp-cursor-portal v0.1.6

Feature release for display aliases and display history in multi-monitor field layouts.

## Assets

- `vp-cursor-portal-setup.exe`: Windows installer that installs under `Program Files`.
- `vp-cursor-portal-win-x64.zip`: self-contained portable Windows x64 build.

## Features

- Adds editable display aliases in the Displays tab.
- Shows aliases in the display preview, layout canvas, Add Display list, layout summaries, and Identify overlays.
- Keeps original Windows display IDs internally for layout matching, monitor rebinding, and cursor routing.
- Tracks detected display history with `Connected` and `Not detected` states.
- Preserves aliasless display rows so previously seen displays remain visible after refreshes and restarts.
- Supports clearing aliases for connected displays and removing not-detected saved display rows.

## Fixes

- Prevents blank display alias rows from being entered in the alias grid.
- Fills missing `Last seen` timestamps when saved aliases are detected again.
- Updates `Last seen` timestamps when previously not-detected aliases reconnect.
- Adds regression coverage for layout snap portal behavior from the preceding field-test branch.

## Compatibility

- Existing configuration files without display aliases continue to load with an empty alias list.
- Saved layouts keep their existing zone IDs and Windows rectangles.
- Cursor routing behavior remains based on original Windows display IDs and coordinates, not aliases.
- Installer or ZIP updates should continue to preserve `%AppData%\vp-cursor-portal\config.json`.

## Validation

- Added app/core tests for display alias config round-trips, alias fallback labels, refresh preservation, aliasless display history, and original zone ID preservation.
- Added tests for missing and stale `Last seen` timestamp handling.
- Run `git diff --check` before release tagging.
- Full local `dotnet test` was not run on the development machine because no .NET SDK is installed there.

## Install Notes

For most users, download and run `vp-cursor-portal-setup.exe`.

Use the ZIP artifact only when you want a portable folder without installer registration.

The installer and executable are not code-signed yet, so Microsoft Defender SmartScreen may show an unknown publisher warning.

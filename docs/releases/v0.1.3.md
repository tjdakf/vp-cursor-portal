# vp-cursor-portal v0.1.3

Patch release for profile execution cleanup after v0.1.2.

## Assets

- `vp-cursor-portal-setup.exe`: Windows installer that installs under `Program Files`.
- `vp-cursor-portal-win-x64.zip`: self-contained portable Windows x64 build.

## Fixes

- Stops the previous cursor-routing layout after a successful H2-only profile execution, without clearing the selected active layout reference.
- Keeps the previous cursor-routing layout running when an H2-only preset command fails, so a failed H2 ACK does not leave the operator with a blank routing state.
- Stabilizes view-model fixture cleanup in tests so asynchronous autosave writes do not intermittently hold `config.json` during CI teardown.

## Validation

- Added regression coverage for successful H2-only profiles stopping active routing.
- Added regression coverage for failed H2-only profiles preserving active routing.
- Confirmed the Windows Build workflow passes for the merged fix branch before release.

## Install Notes

For most users, download and run `vp-cursor-portal-setup.exe`.

Use the ZIP artifact only when you want a portable folder without installer registration.

The installer and executable are not code-signed yet, so Microsoft Defender SmartScreen may show an unknown publisher warning.

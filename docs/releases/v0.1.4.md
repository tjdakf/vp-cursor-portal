# vp-cursor-portal v0.1.4

Patch release for portal-only cursor travel in custom visual layouts.

## Assets

- `vp-cursor-portal-setup.exe`: Windows installer that installs under `Program Files`.
- `vp-cursor-portal-win-x64.zip`: self-contained portable Windows x64 build.

## Fixes

- Prevents Windows' native monitor adjacency from leaking through the outer edge of custom H2 layouts.
- Allows visible-zone-to-visible-zone cursor movement only when the movement matches a configured portal.
- Keeps existing ratio-based portal mapping behavior for valid portal transitions.
- Preserves same-zone movement, hidden-zone rejection, outside-zone rejection, and existing segmented portal behavior.

## Validation

- Added regression coverage for visible-zone-to-visible-zone movement without a matching portal reverting to the last valid cursor position.
- Added regression coverage for same-zone movement staying allowed.
- Added regression coverage confirming configured visible-zone portals still map by ratio.
- Codex review found no issues in the routing change.

## Install Notes

For most users, download and run `vp-cursor-portal-setup.exe`.

Use the ZIP artifact only when you want a portable folder without installer registration.

The installer and executable are not code-signed yet, so Microsoft Defender SmartScreen may show an unknown publisher warning.

# vp-cursor-portal v0.1.2

Patch release for field-test issues found after v0.1.1.

## Assets

- `vp-cursor-portal-setup.exe`: Windows installer that installs under `Program Files`.
- `vp-cursor-portal-win-x64.zip`: self-contained portable Windows x64 build.

## Fixes

- Prevents unintended portal generation between separated displays by no longer auto-attaching all draft layout zones during save.
- Stops loading legacy `config.json` files from beside the executable, so stale portable configs cannot reappear after the AppData config is missing.
- Prevents editable grids from deleting layouts, devices, zones, or portals through implicit DataGrid row deletion.
- Keeps the current active cursor layout running when an H2-only profile executes or when a required H2 ACK fails before a new layout is applied.
- Reduces startup black-screen risk by forcing WPF software rendering and hiding the auto-start tray window only after its first render completes.

## Validation

- Added regression coverage for separated layout zones staying separated during portal generation.
- Added regression coverage for editable DataGrids disabling implicit row deletion.
- Updated profile execution coverage so failed ACK and H2-only profiles no longer clear active routing.

## Install Notes

For most users, download and run `vp-cursor-portal-setup.exe`.

Use the ZIP artifact only when you want a portable folder without installer registration.

The installer and executable are not code-signed yet, so Microsoft Defender SmartScreen may show an unknown publisher warning.
